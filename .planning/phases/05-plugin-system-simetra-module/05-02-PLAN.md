    MetricName: "simetra_heartbeat",
    MetricType: MetricType.Gauge,
    Oids: new List<OidEntryDto>
    {
        new OidEntryDto(
            Oid: HeartbeatOid,
            PropertyName: "beat",
            Role: OidRole.Metric,
            EnumMap: null)

    Source: MetricPollSource.Module)

**ServiceCollectionExtensions.cs** -- Add a new `AddDeviceModules` method:

1. Add `using Simetra.Devices;` to the usings.
2. Add a new public static extension method:
```csharp
/// <summary>
/// Registers all device module implementations as <see cref="IDeviceModule"/> singletons.
/// Must be called before <see cref="AddSnmpPipeline"/> so that
/// <c>IEnumerable&lt;IDeviceModule&gt;</c> is available when DeviceRegistry and
/// DeviceChannelManager resolve.
/// </summary>
public static IServiceCollection AddDeviceModules(this IServiceCollection services)
{
    services.AddSingleton<IDeviceModule, SimetraModule>();

    // Future modules:
    // services.AddSingleton<IDeviceModule, RouterModule>();
    // services.AddSingleton<IDeviceModule, SwitchModule>();

    return services;
}
```

**Program.cs** -- Insert `AddDeviceModules()` call between `AddSimetraConfiguration` and `AddSnmpPipeline`:

Change from:
```csharp
builder.Services.AddSimetraConfiguration(builder.Configuration);
builder.Services.AddSnmpPipeline();
```

To:
```csharp
builder.Services.AddSimetraConfiguration(builder.Configuration);
builder.Services.AddDeviceModules();
builder.Services.AddSnmpPipeline();
```

The order matters: AddDeviceModules() registers IDeviceModule implementations, then AddSnmpPipeline() registers DeviceRegistry and DeviceChannelManager which depend on IEnumerable<IDeviceModule>.
