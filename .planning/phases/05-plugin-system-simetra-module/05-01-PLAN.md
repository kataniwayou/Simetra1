**DeviceRegistry.cs** -- Add `IEnumerable<IDeviceModule> modules` parameter to the constructor alongside the existing `IOptions<DevicesOptions> devicesOptions`:

1. Add `using Simetra.Devices;` to the usings.
2. Change the constructor signature to: `public DeviceRegistry(IOptions<DevicesOptions> devicesOptions, IEnumerable<IDeviceModule> modules)`
3. Keep existing config device registration logic unchanged (the foreach over devicesOptions.Value.Devices).
4. After the config loop, add a second foreach over `modules`:
   ```csharp
   foreach (var module in modules)
   {
       var ip = IPAddress.Parse(module.IpAddress).MapToIPv4();
       var info = new DeviceInfo(module.DeviceName, module.IpAddress, module.DeviceType, module.TrapDefinitions);
       _devices[ip] = info;
   }
   ```
5. Module devices are registered AFTER config devices, so if there is an IP collision, the module takes precedence (module is authoritative for its IP).
6. Update the XML doc on the constructor to mention the `modules` parameter.

**DeviceChannelManager.cs** -- Add `IEnumerable<IDeviceModule> modules` parameter to the constructor alongside the existing parameters:

1. Add `using Simetra.Devices;` to the usings.
2. Change the constructor signature to include `IEnumerable<IDeviceModule> modules` after `IOptions<ChannelsOptions> channelsOptions`.
3. After collecting device names from config, also collect device names from modules:
   ```csharp
   var configDeviceNames = devicesOptions.Value.Devices.Select(d => d.Name);
   var moduleDeviceNames = modules.Select(m => m.DeviceName);
   var allDeviceNames = configDeviceNames.Concat(moduleDeviceNames);
   ```
4. Iterate `allDeviceNames` instead of `devices` in the channel creation loop.
5. The dictionary initial capacity should be the total count: `devices.Count + modules.Count()`. Since `modules` is `IEnumerable`, materialize it first if needed or use a reasonable estimate. Simplest: just use `new Dictionary<string, Channel<TrapEnvelope>>(StringComparer.Ordinal)` without pre-sizing (small collection).
6. Update the XML doc on the constructor to mention the `modules` parameter.
