  <action>
1. Install KubernetesClient NuGet package version 18.0.13:
   `dotnet add src/Simetra/Simetra.csproj package KubernetesClient --version 18.0.13`

2. Create `src/Simetra/Telemetry/K8sLeaseElection.cs`:
   - sealed class implementing both `BackgroundService` and `ILeaderElection`
   - `volatile bool _isLeader` field for thread-safe single-writer-multiple-reader access
   - `bool IsLeader => _isLeader` property
   - `string CurrentRole => _isLeader ? "leader" : "follower"` property
   - Constructor takes: `IOptions<LeaseOptions> leaseOptions`, `IOptions<SiteOptions> siteOptions`, `IKubernetes kubeClient`, `IHostApplicationLifetime lifetime`, `ILogger<K8sLeaseElection> logger`
   - Store unwrapped options values and dependencies as private readonly fields

3. ExecuteAsync implementation:
   - Create `LeaseLock` with: `_kubeClient`, `_leaseOptions.Namespace`, `_leaseOptions.Name`, `_siteOptions.PodIdentity ?? Environment.MachineName`
   - Create `LeaderElectionConfig(leaseLock)` with:
     - `LeaseDuration = TimeSpan.FromSeconds(_leaseOptions.DurationSeconds)`
     - `RetryPeriod = TimeSpan.FromSeconds(_leaseOptions.RenewIntervalSeconds)`
     - `RenewDeadline = TimeSpan.FromSeconds(_leaseOptions.DurationSeconds - 2)`
   - Create `LeaderElector(config)` and wire event handlers:
     - `OnStartedLeading`: set `_isLeader = true`, log Information "Acquired leadership for lease {LeaseName}"
     - `OnStoppedLeading`: set `_isLeader = false`, log Information "Lost leadership for lease {LeaseName}"
     - `OnNewLeader`: log Information "New leader observed: {Leader}"
   - Call `await elector.RunAndTryToHoldLeadershipForeverAsync(stoppingToken)` -- NOT RunUntilLeadershipLostAsync

4. StopAsync override for graceful lease release (HA-05):
   - Call `await base.StopAsync(cancellationToken)` first (cancels the ExecuteAsync stoppingToken)
   - If `_isLeader`, try to delete the lease via `_kubeClient.CoordinationV1.DeleteNamespacedLeaseAsync(_leaseOptions.Name, _leaseOptions.Namespace, cancellationToken: cancellationToken)`
   - Log Information on success: "Lease {LeaseName} explicitly released for near-instant failover"
   - Catch Exception: log Warning "Failed to explicitly release lease {LeaseName} -- followers will acquire after TTL expiry"
   - Set `_isLeader = false` after the try-catch

5. Add XML documentation comments on the class and public members.

IMPORTANT: Use `k8s` namespace for KubernetesClient types (IKubernetes, LeaseLock, LeaderElector, LeaderElectionConfig, KubernetesClientConfiguration). Use `k8s.LeaderElection` for LeaderElector/LeaderElectionConfig. Use `k8s.LeaderElection.ResourceLock` for LeaseLock.

   if (KubernetesClientConfiguration.IsInCluster())
   {
       // Production: Kubernetes Lease-based leader election
       var kubeConfig = KubernetesClientConfiguration.InClusterConfig();
       builder.Services.AddSingleton<IKubernetes>(new Kubernetes(kubeConfig));
       builder.Services.AddSingleton<K8sLeaseElection>();
       builder.Services.AddSingleton<ILeaderElection>(sp =>
           sp.GetRequiredService<K8sLeaseElection>());
       builder.Services.AddHostedService(sp =>
           sp.GetRequiredService<K8sLeaseElection>());

       // Local dev: always leader (single instance)
