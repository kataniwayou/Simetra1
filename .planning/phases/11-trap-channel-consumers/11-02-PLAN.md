Create `ChannelConsumerService` as a `BackgroundService` in `src/Simetra/Services/`.

**Constructor dependencies (all via DI):**
- `IDeviceChannelManager` -- to get readers for all device channels
- `IDeviceRegistry` -- to resolve DeviceInfo by device name for ProcessingCoordinator
- `ISnmpExtractor` -- Layer 3 extraction
- `IProcessingCoordinator` -- Layer 4 processing (metrics + State Vector)
- `ILogger<ChannelConsumerService>` -- structured logging

**ExecuteAsync implementation:**
1. Get all device names from `_channelManager.DeviceNames`
2. For each device name, spawn a consumer task via `ConsumeDeviceChannelAsync(deviceName, stoppingToken)`
3. `await Task.WhenAll(consumerTasks)` to wait for all consumers

**ConsumeDeviceChannelAsync(string deviceName, CancellationToken stoppingToken):**
1. Get reader via `_channelManager.GetReader(deviceName)`
2. Log at Information level: `"Channel consumer started for device {DeviceName}"`
3. `await foreach (var envelope in reader.ReadAllAsync(stoppingToken))` loop:
   - Inside try/catch(Exception):
     - Call `ProcessTrapEnvelope(envelope, deviceName)`
   - Catch block: Log at Error level with device name, correlationId, exception
4. After loop exits, log at Information level: `"Channel consumer completed for device {DeviceName}"`

**ProcessTrapEnvelope(TrapEnvelope envelope, string deviceName):**
1. If `envelope.MatchedDefinition` is null:
   - Log at Debug level: `"Trap without matched definition for {DeviceName}, skipping"`
   - Return
2. Resolve device via `_deviceRegistry.TryGetDeviceByName(deviceName, out var device)`:
   - If not found: Log at Warning level and return
3. Layer 3: `var result = _extractor.Extract(envelope.Varbinds, envelope.MatchedDefinition)`
4. Layer 4: `_coordinator.Process(result, device, envelope.CorrelationId)`
5. Log at Debug level: `"Trap processed for {DeviceName}, definition: {MetricName}, correlationId: {CorrelationId}"`

**TRAP-02 composable middleware requirement:**
The consumer satisfies TRAP-02 by building a consumer-side middleware pipeline using TrapPipelineBuilder. In the constructor, build the consumer pipeline:

```csharp
private readonly TrapMiddlewareDelegate _consumerPipeline;
```

In the constructor:
```csharp
var builder = new TrapPipelineBuilder();
builder.Use(next => async context =>
{
    // Error handling middleware (outermost)
    try
    {
        await next(context);
    }
    catch (Exception ex)
    {
        logger.LogError(ex,
            "Consumer pipeline error for {DeviceName}, correlationId: {CorrelationId}",
            context.Device?.Name ?? "unknown",
            context.Envelope.CorrelationId);
        context.IsRejected = true;
        context.RejectionReason = $"Consumer error: {ex.Message}";
    }
});
builder.Use(next => async context =>
{
    // Logging middleware
    logger.LogDebug(
        "Consumer processing trap for {DeviceName}, definition: {MetricName}, correlationId: {CorrelationId}",
        context.Device?.Name ?? "unknown",
        context.Envelope.MatchedDefinition?.MetricName ?? "unmatched",
        context.Envelope.CorrelationId);
    await next(context);
});
_consumerPipeline = builder.Build();
```

Then in `ProcessTrapEnvelope`, create a TrapContext from the envelope and run it through `_consumerPipeline` before extraction:

```csharp
var trapContext = new TrapContext
{
    Envelope = envelope,
    Device = device
};
_consumerPipeline(trapContext).GetAwaiter().GetResult();
if (trapContext.IsRejected) return;
```

Wait -- the pipeline is async but ProcessTrapEnvelope is sync. Two options:
(a) Make ProcessTrapEnvelope async (cleaner)
(b) Use GetAwaiter().GetResult() (simpler but blocks thread)

Go with option (a): make `ProcessTrapEnvelopeAsync` async. Update `ConsumeDeviceChannelAsync` to `await ProcessTrapEnvelopeAsync(envelope, deviceName)`.

**IMPORTANT anti-patterns to avoid (from research):**
- Do NOT re-stamp correlationId. The envelope already has it from the listener.
- Do NOT catch OperationCanceledException inside the `await foreach` -- let it propagate to end the consumer task naturally.
- Do NOT use .Result or .Wait() -- use async/await throughout.
- Do NOT create one BackgroundService per device -- single service with Task-per-channel.
