<objective>
Implement the METR-01 metric naming convention: change MetricFactory to use PropertyName directly as the OTLP metric name instead of the current `{MetricName}_{PropertyName}` format.

Purpose: The MetricName prefix is redundant because base labels (site, device_name, device_ip, device_type) already provide all the context needed to identify which device and type a metric belongs to. Clean snake_case property names (e.g., `port_rx_octets`, `beat`, `link_state`) produce cleaner, more idiomatic OTLP metrics.

Output: Updated MetricFactory with new naming, all affected tests updated and passing.
</objective>
