**Test 1: TrapEnvelope_FlowsThrough_ConsumerPipeline_ProducesStateVectorAndMetrics**

Steps:
1. Arrange:
   - Build a TrapEnvelope using the SimetraModule's heartbeat definition:
     ```csharp
     var envelope = new TrapEnvelope
     {
         Varbinds = new List<Variable>
         {
             new Variable(
                 new ObjectIdentifier(SimetraModule.HeartbeatOid),
                 new Integer32(1))
         },
         SenderAddress = IPAddress.Loopback,
         ReceivedAt = DateTimeOffset.UtcNow,
         CorrelationId = "e2e-test-corr-1",
         MatchedDefinition = heartbeatDefinition
     };
     ```

2. Act:
   - Write the envelope to the channel: `await channel.Writer.WriteAsync(envelope)`
   - Complete the channel writer: `channel.Writer.Complete()`
   - Start the consumer: `await consumer.StartAsync(CancellationToken.None)`
   - Wait briefly for processing: `await Task.Delay(100)` (channel is completed so consumer finishes fast)
   - Stop the consumer: `await consumer.StopAsync(CancellationToken.None)`

3. Assert:
   - **State Vector entry exists:**
     ```csharp
     var entry = stateVector.GetEntry(simetraModule.DeviceName, heartbeatDefinition.MetricName);
     entry.Should().NotBeNull("State Vector should be updated by consumer pipeline");
     entry!.CorrelationId.Should().Be("e2e-test-corr-1");
     entry.Timestamp.Should().BeCloseTo(DateTimeOffset.UtcNow, TimeSpan.FromSeconds(5));
     ```
   - **Metric recorded with METR-01 naming (PropertyName only):**
     ```csharp
     recordedMeasurements.Should().Contain(m => m.InstrumentName == "beat",
         "metric name should be PropertyName only (METR-01 convention)");
     ```
   - **Metric value correct:**
     ```csharp
     recordedMeasurements.Should().Contain(m => m.InstrumentName == "beat" && m.Value == 1L);
     ```
