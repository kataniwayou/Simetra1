    **Execute method:**
    ```csharp
    public async Task Execute(IJobExecutionContext context)
    {
        // SCHED-08: Read correlationId BEFORE execution
        var correlationId = _correlation.CurrentCorrelationId;
        var jobKey = context.JobDetail.Key.ToString();

        try
        {
            // SCHED-06: OID from SimetraModule (single source of truth)
            var heartbeatOid = SimetraModule.HeartbeatOid;

            var variables = new List<Variable>
            {
                new(new ObjectIdentifier(heartbeatOid), new Integer32(1))
            };

            // Send loopback trap to SNMP listener on 127.0.0.1
            var receiver = new IPEndPoint(IPAddress.Loopback, _listenerPort);

            await Task.Run(() => Messenger.SendTrapV2(
                requestId: 0,
                version: VersionCode.V2,
                receiver: receiver,
                community: new OctetString(_communityString),
                enterprise: new ObjectIdentifier(heartbeatOid),
                timestamp: 0,
                variables: variables));

            _logger.LogDebug(
                "Heartbeat trap sent to 127.0.0.1:{ListenerPort} [CorrelationId: {CorrelationId}]",
                _listenerPort, correlationId);
        }
        catch (OperationCanceledException)
        {
            throw;
        }
        catch (Exception ex)
        {
            _logger.LogError(ex,
                "Heartbeat job {JobKey} failed [CorrelationId: {CorrelationId}]",
                jobKey, correlationId);
        }
        finally
        {
            // SCHED-08: Stamp liveness vector on completion
            _liveness.Stamp(jobKey);
        }
    }
    ```

    **Required usings:**
    ```csharp
    using System.Net;
    using Lextm.SharpSnmpLib;
    using Lextm.SharpSnmpLib.Messaging;
    using Microsoft.Extensions.Logging;
    using Microsoft.Extensions.Options;
    using Quartz;
    using Simetra.Configuration;
    using Simetra.Devices;
    using Simetra.Pipeline;
    ```

    **Important details:**
    - `SimetraModule.HeartbeatOid` is a `public const string` -- direct reference, no DI needed for the module itself
    - The receiver endpoint is `IPAddress.Loopback` (127.0.0.1) on the LISTENER port (SnmpListenerOptions.Port, default 162). This is the trap RECEIVER, not the agent port.
    - `Messenger.SendTrapV2` is synchronous in SharpSnmpLib. Wrap in `Task.Run` to avoid blocking the Quartz thread. Alternatively, if `Messenger.SendTrapV2Async` exists, use that instead. Check SharpSnmpLib API -- per the research doc, `SendTrapV2` is the standard method. Use `Task.Run(() => Messenger.SendTrapV2(...))` for async compatibility.
    - The heartbeat SEND job only proves the scheduler is alive. The trap flowing through the pipeline (listener -> routing -> extraction -> processing) proves the pipeline is alive. These are separate concerns -- do NOT await or verify receipt in this job.
    - The enterprise OID in `SendTrapV2` is set to the heartbeat OID itself. The requestId is 0 (not tracked). Timestamp is 0 (not meaningful for heartbeat).
    - Variable value is `Integer32(1)` -- arbitrary non-zero value indicating a heartbeat "beat."

    ```csharp
    public Task Execute(IJobExecutionContext context)
    {
        var jobKey = context.JobDetail.Key.ToString();

        try
        {
            // SCHED-07: Generate new correlationId
            var newCorrelationId = Guid.NewGuid().ToString("N");
            _correlation.SetCorrelationId(newCorrelationId);

            _logger.LogInformation(
                "Correlation ID rotated to {CorrelationId}",
                newCorrelationId);
        }
        catch (Exception ex)
        {
            _logger.LogError(ex,
                "Correlation job {JobKey} failed", jobKey);
        }
        finally
        {
            // SCHED-08: Stamp liveness vector on completion
            _liveness.Stamp(jobKey);
        }

        return Task.CompletedTask;
    }
    ```

    **Required usings:**
    ```csharp
    using Microsoft.Extensions.Logging;
    using Quartz;
    using Simetra.Pipeline;
    ```

    **Important details:**
    - CorrelationJob is the ONLY scheduled writer of `_correlationId` (via `SetCorrelationId`). The startup code in Program.cs writes the FIRST value before the scheduler starts. After that, only this job writes. Volatile field is safe with single-writer/multiple-reader pattern.
    - The method is synchronous (no async work needed) but returns `Task.CompletedTask` to satisfy the `IJob` interface.
    - CorrelationId format: `Guid.NewGuid().ToString("N")` -- 32-character hex string, no hyphens. Same format used by the startup code in Program.cs.
    - The old correlationId is NOT logged (it's the "previous" one and is no longer relevant). Only the new ID is logged.
    - Note: The correlationId is NOT read before execution for this job specifically because this job IS the one generating the new ID. However, the SCHED-08 requirement says "all jobs read correlationId before execution" -- for CorrelationJob this is a no-op since it immediately replaces the ID. To be fully compliant, you can read the old ID and log it if desired, but it adds no value. The liveness stamp IS required and is in the finally block.
    - Log level is Information (not Debug) because correlation rotation is an important operational event that should be visible in standard logs.
