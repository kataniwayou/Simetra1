<task type="auto">
  <name>Task 2: Create AddSimetraTelemetry extension and update Program.cs DI order</name>
  <files>
    src/Simetra/Extensions/ServiceCollectionExtensions.cs

  </files>
  <action>
    **Step 1: Add `AddSimetraTelemetry` extension method to `ServiceCollectionExtensions.cs`.**

    Add a new method with this signature:
    ```csharp
    public static IHostApplicationBuilder AddSimetraTelemetry(
        this IHostApplicationBuilder builder)
    ```

    Implementation:
    1. Read OtlpOptions from configuration: `var otlpOptions = new OtlpOptions(); builder.Configuration.GetSection(OtlpOptions.SectionName).Bind(otlpOptions);`
    2. Register `ILeaderElection` as singleton `AlwaysLeaderElection`: `builder.Services.AddSingleton<ILeaderElection, AlwaysLeaderElection>();`
    3. Register OpenTelemetry providers:
       ```csharp
       builder.Services.AddOpenTelemetry()
           .ConfigureResource(resource => resource
               .AddService(serviceName: otlpOptions.ServiceName))
           .WithMetrics(metrics => metrics
               .AddMeter(TelemetryConstants.MeterName)
               .AddRuntimeInstrumentation()
               .AddOtlpExporter(o => o.Endpoint = new Uri(otlpOptions.Endpoint)))
           .WithTracing(tracing => tracing
               .AddSource(TelemetryConstants.TracingSourceName)
               .AddOtlpExporter(o => o.Endpoint = new Uri(otlpOptions.Endpoint)));
       ```
    4. Return `builder`.

    **Required usings** (add to the top of ServiceCollectionExtensions.cs):
    - `using OpenTelemetry.Metrics;` (for `AddMeter`, `AddOtlpExporter` on metrics, `AddRuntimeInstrumentation`)
    - `using OpenTelemetry.Resources;` (for `ConfigureResource`, `AddService`)
    - `using OpenTelemetry.Trace;` (for `AddSource`, `AddOtlpExporter` on tracing)
    - `using Simetra.Telemetry;` (for `TelemetryConstants`, `ILeaderElection`, `AlwaysLeaderElection`)

    **IMPORTANT:** `.AddMeter(TelemetryConstants.MeterName)` MUST match exactly what MetricFactory uses: `meterFactory.Create("Simetra.Metrics")`. Using the constant ensures no mismatch (Pitfall 1 from research).

    **IMPORTANT:** Do NOT call `builder.Services.AddMetrics()` -- .NET 9 Web SDK already registers `IMeterFactory` implicitly. `AddOpenTelemetry().WithMetrics()` hooks into the existing registration (Pitfall 6 from research).

    **Step 2: Update `Program.cs` to call `AddSimetraTelemetry` FIRST.**

    Current Program.cs starts with:
    ```csharp
    var builder = WebApplication.CreateBuilder(args);
    builder.Services.AddSimetraConfiguration(builder.Configuration);
    ```

    Change to:
    ```csharp
    var builder = WebApplication.CreateBuilder(args);
    builder.AddSimetraTelemetry();
    builder.Services.AddSimetraConfiguration(builder.Configuration);
    ```

    The telemetry call is on `builder` (IHostApplicationBuilder), NOT `builder.Services` (IServiceCollection), because it needs access to `builder.Configuration` and `builder.Logging`. This is the standard OpenTelemetry hosting pattern.

    Add `using Simetra.Telemetry;` to Program.cs if needed (for the extension method).
